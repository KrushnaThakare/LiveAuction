5. Realtime overlay transport

Primary — WebSocket (target)

  • URL: wss://your-api/ws-overlay-native
  • Topic: /topic/overlay/{tournamentId}/snapshot
  • Latency: ~100–300 ms
  • UI shows: “Live Sync Connected”

Fallback — HTTP poll

  • GET /api/overlay/{id}/snapshot
  • Used when WebSocket fails
  • Latency: ~1.5–3 s
  • Network tab shows many snapshot XHR calls
Health check: DevTools → Network → filter WS (not XHR). You should see ws-overlay-native status 101. Add ?debugOverlay=1 to overlay URL for console logs.

9. Pre-event checklist

Setup (before event)

  • ☐ Create tournament + set squad size
  • ☐ Create teams with budgets & logos
  • ☐ Upload players (Excel) or import registrations
  • ☐ Mark retained players if any
  • ☐ Configure bid rules in Broadcast
  • ☐ Copy all overlay URLs (same tournamentId)
  • ☐ Test WebSocket on LED + OBS PCs
  • ☐ Change default admin password

Live day

  • ☐ Broadcast → Broadcaster mode ON
  • ☐ PC 1: operator logged in → /auction
  • ☐ PC 2: /auction-display fullscreen
  • ☐ PC 3: OBS browser sources loaded
  • ☐ Header says “Live Sync Connected” on displays
  • ☐ Test one SOLD — all screens update together
  • ☐ WhatsApp auto OFF during auction (optional)
